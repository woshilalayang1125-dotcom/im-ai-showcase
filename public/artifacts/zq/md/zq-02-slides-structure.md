# 投委材料 Slides 框架结构 v5

本文件定义 PPT 页序和每页生成策略。正文为通用结构；项目个异性页序放在文末附录。行业分析和公司分析页数不固定，应根据投资逻辑和证据密度自主扩展。

## 1. 输出页序总表

| 序号    | 章节      | 页面类型 | 是否默认生成图 | 作用                  |
| ----- | ------- | ---- | ------- | ------------------- |
| 1     | 封面      | 模板页  | 否       | 项目名称、日期             |
| 2     | 项目背景    | 内容页  | 是       | 为什么出现投资机会           |
| 3     | 项目方案    | 内容页  | 是       | 投资/合作/交易动作          |
| 4     | 核心投资逻辑  | 内容页  | 是       | 3-5 条主逻辑总览          |
| 5     | 行业及市场分析 | 章节页  | 否       | 章节分隔                |
| 6-N   | 行业及市场分析 | 内容页  | 是       | 技术、市场、竞争、产业链、商业模式变化 |
| N+1   | 公司及业务分析 | 章节页  | 否       | 章节分隔                |
| N+2-M | 公司及业务分析 | 内容页  | 是       | 公司、产品、团队、客户、财务、壁垒   |
| M+1   | 投资方案    | 章节页  | 否       | 章节分隔                |
| M+2-K | 投资方案    | 内容页  | 是       | 估值、可比、回报、交易结构       |
| K+1   | 风险及管理   | 章节页  | 否       | 章节分隔                |
| K+2   | 风险及管理   | 内容页  | 是       | 风险项与可控机制            |
| 末页    | Thanks  | 模板页  | 否       | 结尾                  |

## 2. 每页字段标准

每页在本文件中只保留框架级字段：

| 字段 | 写法 |
|---|---|
| 页码 | 最终 PPT 顺序 |
| 章节 | 背景/方案/核心逻辑/行业/公司/投资方案/风险 |
| 标题 | 投委可读的结论或客观主题 |
| 观点灰框 | 1-3 行页面主判断 |
| 主图形式 | 矩阵、流程图、柱线图、分层架构、证据卡、产品截图等 |
| 内容密度 | 中高/高 |
| 生成策略 | 新生成、旧图复用、旧图改造、复制终稿 |
| 数据要求 | 是否含数字、是否需要证据表 |

## 3. 通用页面库

### 3.1 项目背景页

| 字段 | 要求 |
|---|---|
| 适用场景 | 项目机会来自行业变化、客户需求变化、政策窗口、技术拐点 |
| 主图形式 | “变化前后 + 需求痛点 + 项目切入点”三段式 |
| 可见内容 | 行业/客户发生了什么变化，为什么现有方案不够，项目为什么被看见 |

### 3.2 项目方案页

| 字段 | 要求 |
|---|---|
| 适用场景 | 需要解释投资金额、股权、交易路径、合作模式 |
| 主图形式 | 交易结构图、资金流/股权流、方案对比表 |
| 可见内容 | 方案本身、交易条件、集团/基金/合作方角色 |

### 3.3 核心投资逻辑总览页

| 字段 | 要求 |
|---|---|
| 主图形式 | 四象限、四柱、证据栈、逻辑链 |
| 可见内容 | 3-5 条主逻辑，每条必须有项目特异性支撑 |
| 禁止 | 底部泛化“投资关注点”、右侧给研究员看的提示框 |

### 3.4 行业技术入口页

| 字段 | 要求 |
|---|---|
| 主图形式 | 产业链位置图、系统架构图、数据流图 |
| 可见内容 | 该技术/产品处在产业链什么位置，为什么是智能化/效率提升入口 |

### 3.5 行业破局条件页

| 字段 | 要求 |
|---|---|
| 主图形式 | 历史约束 + 新变量 + 可迁移路径 |
| 可见内容 | 过去为什么难，今天哪些变量变化，项目如何利用窗口 |

### 3.6 市场空间页

| 字段 | 要求 |
|---|---|
| 主图形式 | 自上而下测算表 + 柱线图 |
| 可见内容 | 传统口径与新商业模式口径应分开测算；只展示可解释的数据 |

### 3.7 公司基本情况页

| 字段 | 要求 |
|---|---|
| 主图形式 | 公司沿革 + 产品矩阵 + 典型场景/客户 |
| 可见内容 | 历史、主营业务、产品能力、客户场景 |
| 禁止 | “本页聚焦公司基本情况”这类页面说明 |

### 3.8 产品与技术架构页

| 字段 | 要求 |
|---|---|
| 主图形式 | 分层架构、数据结构、能力闭环、产品截图 |
| 可见内容 | 产品为什么支撑核心投资逻辑，数据/算法/工作流如何形成壁垒 |

### 3.9 竞争格局页

| 字段 | 要求 |
|---|---|
| 主图形式 | 二维象限、竞品矩阵、国内/海外/初创分层 |
| 可见内容 | 竞争要素如何变化，项目在哪些维度占优 |
| 禁止 | 简单列“风险与边界”；应写成竞争窗口和护城河成立条件 |

### 3.10 团队页

| 字段 | 要求 |
|---|---|
| 主图形式 | 能力拼图、关键人物履历、能力-产品难点映射 |
| 可见内容 | 团队稀缺性为何与项目难点匹配 |

### 3.11 客户验证页

| 字段 | 要求 |
|---|---|
| 主图形式 | 客户需求变化 + 产品匹配 + 当前推进状态 |
| 可见内容 | 客户真实需求、导入路径、已发生合作、下一阶段交付 |

### 3.12 财务与订单页

| 字段 | 要求 |
|---|---|
| 主图形式 | pipeline结构 + 收入/利润预测柱线图 + 关键指标表 |
| 可见内容 | 订单覆盖率、收入斜率、利润转正、毛利率/费用杠杆 |

### 3.13 可比公司分析页

| 字段 | 要求 |
|---|---|
| 主图形式 | 分层矩阵 + 可比公司实际数据表 |
| 可见内容 | 主锚、交易锚、上沿校验、下限/竞品锚分别统计；给出倍数结论 |
| 禁止 | 把所有样本混算中位数；把高倍数 AI 平台直接套用到早期公司 |

### 3.14 风险及管理页

| 字段 | 要求 |
|---|---|
| 主图形式 | 风险项 + 可控逻辑 + 阶段化落地节奏 |
| 可见内容 | 产品、AI、商业化、交付、治理等风险如何被具体机制降低 |
| 禁止 | “100天观察节点”“后续关注点”等内部跟踪表达，除非用户明确要求 |
