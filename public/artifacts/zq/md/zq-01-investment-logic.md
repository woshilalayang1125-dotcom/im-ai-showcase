# 投委材料投资逻辑框架 v5

本文件用于把“投资逻辑及要点阐述文件”（PDF/Word/访谈纪要/投研讨论稿）转化为投委会可读的投资判断框架。它不是逐页图片提示词，而是三份 md 的总纲：先沉淀投资逻辑、证据口径、章节边界和可见内容标准，再把项目特异性内容放入文末附录。

## 1. 输入与输出

### 1.1 必备输入

| 输入 | 作用 | 处理要求 |
|---|---|---|
| 投资逻辑及要点阐述文件 | 形成核心判断、页序和论证主线 | 提取“事实-推理-结论”，不要照搬过程语言 |
| 项目 datapack | 支撑公司、产品、财务、客户、订单、团队等事实 | 每个数字必须保留来源页/表/行 |
| 访谈纪要/会议纪要 | 补充管理层观点、客户反馈、合作进展 | 区分已发生事实、管理层判断、投资人推断 |
| 外部资料 | 校验行业、市场、竞品、估值、政策等 | 优先官方披露、年报、公告、权威数据库 |
| 既有 PPT/参考图 | 学习最终呈现形式、密度、页序和视觉语言 | 复用结构，不机械照抄内容 |

### 1.2 三份 md 的职责

| 文件 | 主要用途 | 不应承担的内容 |
|---|---|---|
| `01_投委材料投资逻辑框架_通用v5.md` | 总论、核心投资逻辑、章节组织原则、可见内容边界、证据标准 | 不写逐页长提示词 |
| `02_投委材料Slides框架结构_通用v5.md` | 页序、章节、每页标题、观点、主图形式、生成/复用策略 | 不堆详细来源和原文摘录 |
| `03_投委材料逐页内容与证据表_通用v5.md` | 每页可见内容、内部备注、证据索引、数据口径、图片提示 | 不把内部备注直接变成图片文字 |

## 2. 投委会可见内容标准

### 2.1 基本原则

投委材料不是给研究员自己看的过程稿，而是给投资决策者看的结论性材料。页面可见内容必须服务于：

1. 事实总结：项目已经验证了什么。
2. 特异性判断：为什么这个项目不是泛泛的行业机会。
3. 逻辑推演：从事实到投资判断的因果链条。
4. 决策变量：接下来影响是否投、投多少、什么估值的关键变量。

### 2.2 禁止作为可见模块标题的过程语言

以下内容可以出现在 md 的内部备注、证据表、speaker notes 或工作清单里，但不能作为图片/PPT可见模块：

| 禁止可见表达 | 正确处理方式 |
|---|---|
| 价值实现路径 | 改成项目本身的“业务闭环/客户导入/商业化节奏/产品能力演进” |
| 反证信号 | 改成项目特定的“必须证明的能力/导入门槛/失效条件” |
| 边界与判断、风险与边界 | 改成事实化的“可控机制/导入顺序/阶段化约束” |
| 重要平衡点、投资关注点 | 融入结论、表格、矩阵或关键变量，不单独成块 |
| 本页聚焦、本页展示、建议关注 | 直接删除；投委页不需要给读者解释页面用途 |
| 数据来源、证明材料、datapack、本地路径 | 放入 md 证据区，图片中不展示 |

判断标准：如果一句话可以套到任何项目上，它通常不应该出现在图片里；必须改写为本项目的事实、阈值或因果关系。

## 3. 章节组织原则

### 3.1 固定章节与弹性页数

CATL风格投委材料通常包含：

| 章节 | 默认作用 | 页数原则 |
|---|---|---|
| 项目背景 | 为什么现在看这个项目 | 默认 1 页，复杂项目可扩展 |
| 项目方案 | 交易方案、合作方案或投资动作 | 默认 1 页 |
| 核心投资逻辑总览 | 用 3-5 条逻辑收敛全篇 | 默认 1 页 |
| 行业及市场分析 | 行业变化、技术路径、市场空间、竞争格局 | 自主选择页数，证据足则拆页 |
| 公司及业务分析 | 产品、技术、团队、客户、商业模式、财务 | 自主选择页数，不强行压缩 |
| 投资方案 | 交易结构、估值、回报测算、可比公司 | 根据交易复杂度选择页数 |
| 风险及管理 | 风险项、可控机制、落地节奏 | 默认 1 页，重大风险可拆页 |

行业分析和公司分析不设固定页数。页数由“投资逻辑需要证明的命题数量”决定：每页只承载一个主要判断，复杂技术/市场/竞争/财务逻辑必须拆开。

### 3.2 页标题要求

标题必须是可被投委直接理解的逻辑结论或客观主题，不是 Excel 关键信息的复述。

| 低质量标题 | 更合适标题 |
|---|---|
| 子虔在云化与意图数据两个维度位于更适合AI CAD的象限 | AI CAD胜负手转向云化和意图数据，子虔两项基础能力同时占优 |
| 可比公司分析 | 子虔估值应以工业设计软件为主锚，以AI平台上沿校验 |
| 财务预测 | 在手订单覆盖 2026E 目标较高，盈利拐点取决于订单兑现和交付成本 |

## 4. 核心投资逻辑写法

核心投资逻辑总览页一般使用 3-5 条主逻辑，每条由“行业变化/公司特异性/验证证据/投资含义”构成。

推荐结构：

| 主逻辑 | 内容要求 |
|---|---|
| 行业或技术窗口 | 为什么现在出现结构性机会，天花板是否变化 |
| 公司壁垒 | 公司的架构、数据、团队、客户、生态位为何难复制 |
| 商业化/客户验证 | 客户为什么需要、是否愿意导入、导入路径是否现实 |
| 长期平台价值 | 如果短期验证通过，长期可扩展到什么系统/数据/生态 |
| 交易与回报 | 估值、份额、回报、战略价值是否匹配 |

核心逻辑页必须避免“行业空间大、团队优秀、客户资源好、政策支持”等泛化表达；每条都要写出项目特异性证据。

## 5. 数据与证据标准

### 5.1 所有数字必须可追溯

每个数字必须具备以下之一：

1. 本地文件来源：文件名 + 页码/章节/Sheet/行列。
2. 外部来源：链接 + 发布方 + 发布时间/访问日期。
3. 测算逻辑：公式 + 输入变量 + 假设来源 + 结果。

禁止使用“看起来合理”的数字。若无法确认来源，应标注为待核查，不进入图片。

### 5.2 图片中不展示来源，md 中必须展示来源

投委图片可展示数字和结论，但不展示长链接、本地路径、证据管理标签。证据统一放入逐页内容 md 的“证据索引”。

证据表字段：

| 字段 | 要求 |
|---|---|
| claim_id | 稳定编号 |
| claim | 使用的事实或数据 |
| value | 数字/文本 |
| source_type | local_pdf/local_docx/local_xlsx/web/calculation/interview |
| source_detail | 文件页码、Sheet行列或URL |
| calculation_logic | 测算公式与假设 |
| confidence | high/medium/low |
| slide_usage | 使用页码 |
| visible_or_notes | 图片可见或仅内部备注 |

## 6. 估值与财务页原则

### 6.1 财务页

财务页展示“已经发生/公司预测/投资人测算”的关系：

| 模块 | 可见内容 |
|---|---|
| 订单或pipeline | 总额、覆盖率、行业结构、头部客户集中度 |
| 收入与利润预测 | 年份、收入、利润、毛利率、CAGR或增速 |
| 关键财务判断 | 收入兑现依赖项、利润转正年份、成本/交付杠杆 |

不展示“预测不等于估值依据”“重要提示”等提醒研究员的话。

### 6.2 可比公司页

可比公司必须分层，不把所有公司混在一个中位数里。

| 层级 | 用途 |
|---|---|
| 主估值锚 | 与项目商业模式/产品形态最接近的上市公司，进入基准倍数 |
| 交易锚 | 并购或融资案例，用于战略价值校验，不进入上市倍数中位数 |
| 上沿校验 | 高倍数平台型公司，只解释叙事上限，不进入基准 |
| 下限/竞品锚 | 国内替代或弱可比公司，用于下限和竞争格局 |

若公司仍处投入期、利润率不稳定，优先用远期收入 PS/ARR；PE 仅用于成熟态交叉验证。

## 7. 图片生成提示原则

图片提示词必须只包含可见内容，不把 md 的内部备注带入图中。

生成前检查：

1. 标题是否是投委可读的结论或客观主题。
2. 灰框是否是页面观点，而非“本页聚焦”。
3. 是否包含禁用过程标签。
4. 是否有无来源数字。
5. 是否有空泛口水话。
6. 是否保持中高密度但不牺牲可读性。
7. 是否没有 CATL logo/page-corner mark，除非内容确需。

1. AI 重构 CAD 价值：从建模工具变为可被需求、规则库和模型调用的制造业软件基础设施。

### 8.3 子虔项目禁用倾向

1. 不把项目定位为单纯“国产3D CAD替代”。
2. 不把 Vibe CAD 市场只按 SaaS seat/license 口径估算。
3. 不把 Palantir/ServiceNow 等 AI 平台倍数直接纳入基准中位数。
4. 不把风险页写成给研究员看的观察清单；应写成产品试点、AI闭环、商业化体系的可控机制。
